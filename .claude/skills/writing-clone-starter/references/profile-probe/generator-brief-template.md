# Profile Probe｜生成侧模板

## 用途

给生成侧一个统一 brief，避免每次 blind probe 都临时重写要求。

## 推荐结构

### 1. 任务

- 用当前 `<author-id>` profile 生成一篇 fresh probe 正文

### 2. 题目

- 写清本轮 probe 题目

### 3. 输出路径

- `response.md`
- 可选：`run_summary.md`

### 4. 必须体现的动作链

- 高识别入口
- 普通人判断翻译
- 主故事链
- 现实收益 / 现实代价落点
- 可传播收束 + 动作入口

### 5. 禁止事项

- 不读历史评分稿
- 不优化旧文章
- 不把系统说明写进正文
- 不靠词包硬装像

### 6. 本轮特殊修复目标

- 只写 1–3 条
- 来自上一轮失分项

## 关键提醒

生成侧只负责：**出 probe**。

它不负责：

- 解释为什么像
- 自己给自己打分
- 定义是否过线
