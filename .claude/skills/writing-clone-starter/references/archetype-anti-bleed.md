# 强文章原型｜Anti-Bleed 规则

## 目的

这份文件专门防止强文章模式串味。

不是禁止写好，而是禁止为了显得高级，偷借别的模式或坏写法的气质捷径。

## 1. 观点拆解型

### 最容易串味

- 串成 Dan Koe 式 worldview 高拟态

### 禁止项

- 不要默认用趋势宣告 + 地图 / 拼图 / 主权这一套作者信号包开场
- 不要为了抬气质，把中段突然拔到作者式愿景宣言
- 不要拿作者口吻代替机制支撑

## 2. 方法教程型

### 最容易串味

- 串成教培讲义或产品说明书

### 禁止项

- 不要写成“第一步、第二步、第三步”流水账交代
- 不要只给步骤，不解释为什么这样排
- 不要把正文写成客服文档口吻

## 3. 案例复盘型

### 最容易串味

- 串成新闻复述

### 禁止项

- 不要把篇幅都花在复述事实
- 不要只靠故事性推着走，却不给机制提炼
- 不要为了显得深刻，硬从个例跳到空泛大道理

## 4. 认知升级型

### 最容易串味

- 串成空洞大词文

### 禁止项

- 不要只做概念拔高，不把新框架压回具体问题
- 不要靠 worldview 大词制造“有深度”的错觉
- 不要把高拟态作者信号包当作认知升级型的捷径

## 总提醒

- 强文章模式可以强，但不能偷偷写成作者拟态
- 原型的高级感，来自正文合同成立，不来自信号词密度
