# 强文章原型路由

## 只回答一个问题

给定一个题目，强文章模式优先走哪个原型。

## 路由顺序

按下面顺序判断，命中更靠前的就先走：

### 1. 方法教程型

命中信号：

- 题目在问“怎么做 / 如何开始 / 具体步骤”
- 读者读完最需要的是可执行路径
- 文章核心价值来自动作顺序而不是判断辩论

### 2. 案例复盘型

命中信号：

- 用户给了具体案例、项目、事件或人物
- 文章价值来自“从案例看机制”
- 主题需要先讲发生了什么，再提炼为什么这样

### 3. 观点拆解型

命中信号：

- 题目自带 stance
- 更像“为什么该重新判断这件事”
- 文章核心是拆旧理解、立新判断

### 4. 认知升级型

命中信号：

- 题目核心是换框架、换坐标系
- 比起动作步骤，更需要新的理解方式
- 更像“看懂这件事该站在哪一层理解”

## 冲突时怎么判

### 方法 vs 观点

- 如果读者最需要的是动作顺序，优先方法教程型
- 如果读者最需要的是判断翻转，优先观点拆解型

### 案例 vs 观点

- 如果案例只是引子，正文主要在立观点，优先观点拆解型
- 如果案例本身就是正文骨架，优先案例复盘型

### 认知 vs 观点

- 如果文章最终要落成“应该相信什么”，优先观点拆解型
- 如果文章最终要落成“应该怎样理解同类问题”，优先认知升级型

## 默认策略

- 能明确路由时，不混原型
- 只有在用户明确要求“兼顾方法和观点”时，才允许以一个主原型 + 一个副原型辅助，但正文合同仍以主原型为准
- 拿不准时，先问：这篇文章最重要的是让读者改变判断、学会方法、看懂案例，还是升级理解框架
