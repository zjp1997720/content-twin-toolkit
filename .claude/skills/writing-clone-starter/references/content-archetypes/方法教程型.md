# 强文章原型｜方法教程型

## 这类原型解决什么问题

适合用户想把一个方法、流程、操作路径讲清楚，让新手能直接开始。

它的关键不是“步骤越多越好”，而是：

- 把适用边界讲清
- 把动作顺序讲清
- 把第一步压到足够可执行

## 什么时候优先用

- 用户在问“怎么做 / 怎么开始 / 如何从 0 到 1”
- 主题本身更适合按动作顺序推进
- 读者读完后最需要的是一个能照着走的起步动作

## 正文合同

### 开头怎么进

- 先界定问题与适用边界
- 说清这套方法解决什么，不解决什么
- 开头要降低复杂度，而不是上价值

### 中段怎么推进

- 按动作顺序讲步骤，但不要写成说明书
- 每一步都要回答：为什么有这一步、常见误区是什么、做对后会发生什么
- 可以用前后对比、易错点和最小案例把步骤压实

### 结尾怎么收

- 收成一个可执行的起步动作
- 可以提醒边界和节奏，但不要变成“课后作业布置”
- 读者读完要知道第一步该怎么动，而不是只知道这套流程很完整

## 最容易写坏成什么样

- 写成讲义或产品说明书
- 步骤很多，但没有主线和优先级
- 只给动作，不解释动作背后的判断

## 素材库怎么取

- 优先取：步骤框架、操作清单、易错点、前后对比
- 次优先：一个足够小的案例，用来让步骤落地
- 不要把素材库当知识堆栈，重点是帮读者顺着走完路径

## 最容易串味到哪里

- 最容易串成教培讲义

## 原型提醒

- 教程型也要有高手感，但高手感来自“讲得明白”，不是来自高概念词密度
- 如果正文开始不断拔高价值观，先检查是不是已经偏离教程型合同
