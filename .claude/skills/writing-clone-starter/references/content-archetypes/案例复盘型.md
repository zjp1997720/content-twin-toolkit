# 强文章原型｜案例复盘型

## 这类原型解决什么问题

适合用一个案例、一次事件、一个项目，带读者从现象走到机制，再提炼可迁移判断。

它的关键不是“把故事讲完整”，而是：

- 为什么这个案例值得看
- 这个案例真正说明了什么
- 读者能迁移走什么判断

## 什么时候优先用

- 用户给了具体案例、项目、事件或人物
- 文章目标是“从案例里看出机制”
- 读者需要的不是观点喊话，而是对一个真实样本的拆解

## 正文合同

### 开头怎么进

- 先交代这个案例为什么值得看
- 可以快速给结果或关键转折，但不要把开头写成新闻导语
- 读者要尽快知道：这不是八卦复述，而是一次有判断的复盘

### 中段怎么推进

- 顺序优先：现象 -> 关键转折 -> 机制解释 -> 可迁移判断
- 复盘的重点不是多细，而是要把“发生了什么”和“为什么会这样”接起来
- 至少要有一次从事实层走到结构层的抽象

### 结尾怎么收

- 提炼一个能迁移的判断或原则
- 不要只用“这个案例很有启发”这种空收尾
- 让读者知道这个案例以后该如何被复用

## 最容易写坏成什么样

- 变成故事转述，没有机制提炼
- 信息很多，但看不出主张
- 强行拔高成大道理，反而失去案例的真实力量

## 素材库怎么取

- 优先取：案例经过、关键转折、结果、机制解释
- 次优先：一两个对照样本，用来说明这不是偶然
- 不要堆太多枝节素材，重点是围绕关键转折压实机制

## 最容易串味到哪里

- 最容易串成新闻复述

## 原型提醒

- 案例复盘型最怕“有故事没判断”
- 如果中段一直在补事实，却没有把事实组织成解释链，这篇就还没进入复盘型正文合同
