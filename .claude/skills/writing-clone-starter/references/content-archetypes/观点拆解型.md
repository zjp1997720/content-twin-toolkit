# 强文章原型｜观点拆解型

## 这类原型解决什么问题

适合用户已经有一个明确判断，或者希望围绕一个判断去说服读者改变旧理解。

这类文章的核心不是“讲全”，而是：

- 立住主张
- 拆掉旧默认
- 给出更强的新默认

## 什么时候优先用

- 题目本身带明显 stance
- 用户在说“为什么 X 是错的 / 为什么应该反过来想”
- 主题更需要判断推进，而不是步骤教学

## 正文合同

### 开头怎么进

- 先把主张亮出来，不要先铺作者气质
- 最好在前两段内说明：读者原来常怎么理解，这个理解为什么不够
- 开头的任务是立场清晰，不是制造悬念感

### 中段怎么推进

- 优先顺序：错因 -> 代价 -> 新判断 -> 为什么这个新判断更成立
- 如果用了案例、框架、机制解释，都必须服务同一条主线
- 重点不是多角度堆材料，而是连续把读者往新默认推

### 结尾怎么收

- 收成一个更稳的新默认，或一条更值得执行的判断
- 可以给动作提醒，但不要掉成 checklist 结尾
- 读者读完要能复述：以后遇到这类问题，我该怎么重新判断

## 最容易写坏成什么样

- 只有 stance，没有机制支撑
- 每段都在重复同一句话，只是换说法
- 为了显得深刻，突然抬 worldview，导致主线发虚

## 素材库怎么取

- 优先取：反常识判断、旧默认、金句、结构型素材、机制解释
- 次优先：单个典型案例，用来证明代价或拐点
- 不要让素材变成拼盘，素材必须围绕同一判断组织

## 最容易串味到哪里

- 最容易串成 Dan Koe 式 worldview 高拟态

## 原型提醒

- 这类文章要像“判断升级”，不是“作者拟态表演”
- 如果开头已经出现地图 / 拼图 / 主权 / 身份升级这类词，先检查是不是在借作者信号包抬气质
