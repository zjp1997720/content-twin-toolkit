# 强文章模式共同标准

这份标准只回答一件事：

> **不管像不像某个作者，这篇文章本身强不强。**

## 一票否决项

命中任一条，直接判定不通过：

1. 只有正确废话，没有具体判断
2. 结构松散，读完说不出主线
3. 只有素材拼贴感，没有真正推进
4. 开头、正文、结尾互相脱节
5. 新手零个人素材时，文章明显发空

## 五项核心标准

### 1. 主张是否站得住

- 必须有清晰 stance
- 不能只是中性整理
- 读完能复述作者到底想让人相信什么

### 2. 结构是否能带人走下去

- 开头必须给进入理由
- 中段必须连续推进，而不是换说法重复
- 结尾必须收成判断、动作或新的认知出口

### 3. 内容是否不空

- 至少要有可感知的论点、案例、框架或机制解释
- 不能只有抽象口号
- 素材库介入后，应该补强，不应压垮主线

### 4. 表达是否有高手感

- 不写成讲义
- 不写成流水账
- 不写成模板化 AI 汇报
- 读感要紧，推进要快，信息密度要够

### 5. 是否对新手友好

- 没有长期沉淀的用户也能跑通
- 不把个人经历当成默认必要条件
- 即使只有一个主题，也能成稿

## 正文合同（surface plane）

这份 rubric 不只是判断“质量行不行”，还要约束强文章模式**正文应该长什么样**。

### 1. 开头合同

- 先立主张，不先做作者式世界观表演
- 让读者快速知道：这篇文章到底要说服我相信什么
- 不默认用“趋势宣告 + 地图/拼图/主权”这一套作者拟态包开场

### 2. 推进合同

- 重点是把问题讲明白，而不是把气质做得像谁
- 中段要不断推进主张，不靠高概念堆叠制造高级感
- 如果用了框架、案例、机制解释，它们都应该服务同一条主线

### 3. 结尾合同

- 收成判断、动作、提醒或新的认知出口
- 不默认上升到作者式愿景宣言
- 让读者知道下一步该怎么想、怎么做，或为什么这件事重要

## 防串味约束

强文章模式最容易犯的错，不是写差，而是默认滑向高拟态作者包。

### 默认不要把这些当成气质捷径

- 地图 / 拼图 / 主权 / 身份升级 / 复利
- “旧默认失效”式作者宣告
- 过强的 worldview 压顶

不是完全禁止这些词或这些思路，而是禁止：

> 为了显得高级，默认把强文章模式写成 Dan Koe 式高拟态稿。

## 通过标准

至少同时满足：

1. 主张成立
2. 结构成立
3. 内容不空
4. 读感不假
5. 新手可跑通

## 失败时优先检查什么

1. 主张不清
2. 结构没压紧
3. 素材多但没有被组织成主线
4. 表达像 AI 在交差
5. 正文明显串味到作者拟态模式
