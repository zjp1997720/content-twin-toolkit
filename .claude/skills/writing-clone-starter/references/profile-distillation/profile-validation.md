# 人物 Profile 蒸馏｜验证协议

## 目的

这份文件负责判断：

- 一个新人物 profile 是否真的合格
- 它是否只是“看起来像”，还是已经具备独占性、生成力和边界意识

它验证的是 **profile 资产**，不是单篇文章调参。

## 最小四步验证

### 1. 已知样本重构验证

问题：

> 用该人物明确写过 / 说过的题材，profile 是否能逼近其结构与动作？

检查点：

- 能否看出熟悉的开头动作
- 能否看出稳定的重定义方式
- 能否看出该人物的结尾收束习惯

### 2. 未知题材生成验证

问题：

> 用此人未明确讨论过、但仍在带宽内的话题，profile 是否仍有生成力？

检查点：

- 不只是复读旧样本
- 能在新问题上继续保持同一路径感

### 3. 超带宽边界验证

问题：

> 用明显弱带宽题材时，profile 是否会表现出诚实边界？

检查点：

- 是否能识别超带宽
- 是否会降级或至少停止硬拟态

### 4. 盲比区分度验证

问题：

> 与普通强稿 baseline 对照时，差异是否主要落在正文，而不是说明层？

检查点：

- 是否能看出动作链差异
- 是否避免掉成公共高概念词堆砌

## 最低通过条件

一个新人物 profile 至少要满足：

1. 已知样本重构能看到结构与动作接近
2. 未知题材仍有可感知生成力
3. 超带宽时不会继续伪拟态
4. 与 baseline 的差异主要落在正文层

## 失败后如何处理

- 如果问题是特征虚 → 回到 `extraction-gates.md`
- 如果问题是证据弱 → 回到 `research-layers.md` 和 `source-ledger.md`
- 如果问题是素材层断开 → 回到 `material-sync.md`

不要一上来重写全部 profile，先找失败落在哪一层。
