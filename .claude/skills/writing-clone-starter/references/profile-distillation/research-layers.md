# 人物 Profile 蒸馏｜统一调研分层

## 目的

这份文件负责把“蒸馏一个人物”时要收哪些证据、先走哪条链、失败后怎么补链，固定成 starter 的维护者协议。

它不负责直接生成 profile 正文，只负责把输入材料准备成可蒸馏状态。

## 总原则

- 先拿到**足够可靠的材料**，再提炼
- 先把材料**落到 Obsidian / 本地**，再进入 profile 蒸馏
- 优先使用已验证的上游工具链，但不要把工具链写死成唯一合法路径
- 真正写死的是结果：证据层齐、素材层齐、后续可追溯

## 优先工具链

默认优先按下面顺序走：

1. 侦察模块确定人物与来源范围
2. 用 `web-clipper` 把公开文章抓到本地 / Obsidian
3. 用 `content-goldmine-gemini` 把本地文章拆进素材库
4. 再进入 profile 蒸馏与验证

### 为什么这样排

- `web-clipper` 解决“先把材料稳定抓下来”
- `content-goldmine-gemini` 解决“把材料稳定拆进素材库”
- profile 蒸馏模块解决“把人物风格提炼成 starter 可消费资产”

## 弹性成功路径

如果优先工具链失败，允许 agent 自主换路，但必须满足同样的终局条件：

1. 人物相关材料被稳定拿到
2. 高价值素材被同步落进素材库
3. 正式 profile 特征能回链到来源与过滤理由

### 常见换路场景

- 网页抓取失败 → 改走 browser / transcript / 手动 source 收集
- 文章不是网页正文 → 改走现成 Markdown、PDF、转录稿
- 批量拆解质量不够 → 改走人工筛选 + 定向拆解

## 五层证据分层

### 1. 长文与著作层

负责支撑：

- worldview
- 价值排序
- 系统性判断

优先来源：

- 著作
- 长文 newsletter
- 长篇博客 / essay
- 系统性访谈稿

### 2. 长对话与访谈层

负责支撑：

- 即兴重定义
- 临场推理
- 回答问题的方式
- 被追问时的稳定动作

优先来源：

- 播客 transcript
- 深度访谈
- AMA / 长视频转录

### 3. 表达碎片层

负责支撑：

- 词汇偏好
- 句法节奏
- 命名方式
- 标题 / 开头 / 金句风格

优先来源：

- 社交媒体原文
- 短帖
- 短文片段
- 已拆进素材库的标题 / 开头 / 金句

### 4. 决策与行为层

负责支撑：

- 人物到底怎样判断
- 说法和做法的一致 / 不一致
- 关键转折与取舍

优先来源：

- 公开决策记录
- 产品/投资/转型案例
- 时间线中的关键行动

### 5. 外部评价与批评层

负责支撑：

- 盲区
- 争议点
- 反样本
- 易误判风险

优先来源：

- 权威评论
- 批评视角
- 同行对照

## 最低完成条件

进入提炼前，至少要满足：

- 长文与著作层：有一级主证据
- 长对话与访谈层：有能看出即兴推理的材料
- 表达碎片层：有足够支撑标题 / 开头 / 高频动作的材料
- 决策与行为层：至少能看到 1-2 个关键取舍
- 外部评价与批评层：至少能识别主要争议或误判点

## 输出要求

调研阶段的完成，不等于只收集 URL。

至少要同时产出两类东西：

1. **蒸馏证据底稿**：供 worldview / moves / signals 提炼
2. **素材库资产**：供 starter 后续写作直接调用

如果只完成第 1 类，没有第 2 类，这轮调研仍然不算闭环。
