# 人物 Profile 蒸馏｜三重过滤门

## 目的

这份文件负责决定：哪些候选特征能进入正式 profile，哪些只能降级成观察，哪些应该丢弃。

它的作用，是压住三类常见误判：

- 公共词表误判成作者独占性
- 通用强稿习惯误判成人物风格
- 单次漂亮表达误判成稳定动作

## 候选特征分类

### 1. 认知层候选

- worldview
- 判断框架
- 价值排序
- 解释世界的默认坐标

### 2. 结构层候选

- 开头动作
- 重定义路径
- 中段推进方式
- 结尾收束方式

### 3. 语言层候选

- 句法偏好
- 抽象度
- 比喻密度
- 高频表达与禁忌表达

## 三重过滤门

### 门 1：跨场景复现

问题：

> 这条特征是否在多个题材、多个来源、多个场景中反复出现？

通过信号：

- 不只出现在一篇爆文里
- 不只出现在一句名言里
- 能在长文、对话、表达碎片中交叉出现

### 门 2：生成力

问题：

> 这条特征是否能指导新文章生成，而不是只能解释旧文本？

通过信号：

- 用它能推断该人物面对新题材大概率会怎样写
- 它能约束结构、判断或语言，而不是只做“总结标签”

### 门 3：排他性

问题：

> 这条特征能把该作者和“普通强稿”区分开吗？

通过信号：

- 不是所有高手都会这么写
- 不是只因为“文章写得好”就成立
- 与 baseline 对照时，能构成真实差异

## 过门规则

- **三门都过** → 进入正式 profile 资产
- **只过 1-2 门** → 降级为补充观察，不能进入核心资产
- **0 门未过** → 直接丢弃

## 硬约束

- 所有进入正式资产的特征，必须同时进入 `source-ledger.md`
- 没有 ledger 记录的特征，不算正式通过

## 常见误判示例

### 假通过 1：公共高概念词

- “system / map / leverage / sovereignty”

为什么不够：

- 可能只是公共聪明人词汇，不足以证明作者独占性

### 假通过 2：单篇爆款句式

- 某篇文章里一句极强标题或金句

为什么不够：

- 可能只是一次命中，不代表稳定风格

### 假通过 3：通用好文结构

- 开头立冲突、中段讲机制、结尾落判断

为什么不够：

- 这是强写作常识，不等于该人物独特 profile

## 结果导向提醒

这份门槛文件不是为了“多收特征”，而是为了：

- 少收，但收得准
- 宁缺，别滥
- 让正式 profile 真有独占性和生成力
